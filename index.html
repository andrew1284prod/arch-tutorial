Шаг 1: Проверка интернет-подключения

Перед началом установки необходимо убедиться, что у вас есть активное интернет-подключение. Arch Linux загружает пакеты из онлайн-репозиториев во время установки.

Если у вас проводное соединение, оно должно работать автоматически. Для беспроводного подключения используйте утилиту iwctl.

Проверка подключения 
# Проверка доступности сетевых ресурсов
ping archlinux.org -c 4

# Или альтернативный вариант
ping google.com -c 4
Совет: Если команда ping не работает, это может означать, что у вас нет сетевого подключения или ICMP-трафик заблокирован. Попробуйте использовать curl -I https://archlinux.org для проверки.

Для беспроводного подключения:

Настройка Wi-Fi 
# Запуск интерактивной утилиты для Wi-Fi
iwctl

# Внутри iwctl:
[iwd]# device list           # Показать доступные сетевые устройства
[iwd]# station wlan0 scan    # Просканировать сети (замените wlan0 на ваше устройство)
[iwd]# station wlan0 get-networks  # Показать найденные сети
[iwd]# station wlan0 connect "SSID"  # Подключиться к сети (укажите имя сети)

# Выйти из iwctl
[iwd]# exit

Шаг 2: Разметка диска

Определите целевой диск и создайте разделы. Для UEFI-системы вам понадобится как минимум два раздела: EFI System Partition (ESP) и корневой раздел.

ВАЖНО: Убедитесь, что выбрали правильный диск, так как все данные на нём будут удалены!

Определение дисков 
# Показать все диски и разделы
lsblk

# Более подробная информация
fdisk -l
Внимание: Будьте осторожны при выборе диска. Если у вас несколько дисков, они могут называться /dev/sda, /dev/sdb и т.д. В последних системах могут использоваться названия /dev/nvme0n1.
Разметка диска с помощью cfdisk 
# Запуск cfdisk для нужного диска (например, /dev/sda)
cfdisk /dev/sda

Создание разделов:

  • EFI System Partition (ESP): Размер 300-512 МБ, тип EFI System
  • Корневой раздел (/): Размер от 60 ГБ (рекомендуется), тип Linux filesystem
Совет: Вы также можете создать отдельные разделы для /home (ваши пользовательские файлы) и swap (файл подкачки). Однако для минимальной установки достаточно только ESP и корневого раздела.

Шаг 3: Форматирование разделов

После создания разделов необходимо отформатировать их в соответствующие файловые системы.

Форматирование ESP в FAT32 
# Замените sdX1 на ваш EFI-раздел (например, /dev/sda1)
mkfs.fat -F32 /dev/sdX1
Форматирование корневого раздела в ext4 
# Замените sdX2 на ваш корневой раздел (например, /dev/sda2)
mkfs.ext4 /dev/sdX2
Совет: Вместо ext4 вы можете использовать другие файловые системы, такие как Btrfs, XFS или ext4 с шифрованием. Однако для новичков рекомендуется ext4 как наиболее стабильный вариант.

Шаг 4: Монтирование разделов

Теперь необходимо примонтировать созданные разделы, чтобы начать установку системы.

Монтирование корневого раздела 
# Монтируем корневой раздел в /mnt
mount /dev/sdX2 /mnt
Создание директории и монтирование ESP 
# Создаем директорию для ESP
mkdir -p /mnt/boot/efi

# Монтируем ESP
mount /dev/sdX1 /mnt/boot/efi
Совет: Если вы создали дополнительные разделы (например, для /home), их также нужно отформатировать и смонтировать в соответствующие директории внутри /mnt.

Шаг 5: Установка базовых пакетов

Теперь установим минимальный набор пакетов Arch Linux в смонтированную систему.

Минимальная установка 
# Установка базовых пакетов, ядра Linux и микрокода
pacstrap /mnt base linux linux-firmware
Расширенная установка (рекомендуется) 
# Установка с дополнительными полезными пакетами
pacstrap /mnt base base-devel linux linux-firmware \
          networkmanager nano sudo grub efibootmgr \
          dhcpcd iwd bash-completion
Совет: Если у вас процессор Intel или AMD, вы можете добавить соответствующий микрокод для обновлений безопасности: intel-ucode для Intel или amd-ucode для AMD.

Шаг 6: Настройка системы

После установки пакетов необходимо выполнить базовую настройку системы.

Генерация fstab и переход в chroot 
# Генерация таблицы файловых систем
genfstab -U /mnt >> /mnt/etc/fstab

# Проверка сгенерированного fstab
cat /mnt/etc/fstab

# Переход в установленную систему
arch-chroot /mnt

Базовая настройка:

Настройка времени, локали и hostname 
# Установка часового пояса (например, Europe/Moscow)
ln -sf /usr/share/zoneinfo/Europe/Moscow /etc/localtime
hwclock --systohc

# Настройка локали
sed -i 's/#en_US.UTF-8 UTF-8/en_US.UTF-8 UTF-8/' /etc/locale.gen
sed -i 's/#ru_RU.UTF-8 UTF-8/ru_RU.UTF-8 UTF-8/' /etc/locale.gen
locale-gen

echo "LANG=en_US.UTF-8" > /etc/locale.conf

# Установка hostname (имени компьютера)
echo "archlinux" > /etc/hostname

# Добавление hostname в hosts
echo "127.0.0.1 localhost" >> /etc/hosts
echo "::1       localhost" >> /etc/hosts
echo "127.0.1.1 archlinux.localdomain archlinux" >> /etc/hosts
Создание пользователя и настройка sudo 
# Установка пароля для root
passwd

# Создание обычного пользователя (замените username на ваше имя)
useradd -m -G wheel -s /bin/bash username
passwd username

# Настройка sudo для группы wheel
echo "%wheel ALL=(ALL:ALL) ALL" >> /etc/sudoers

Шаг 7: Установка загрузчика

GRUB - самый распространённый загрузчик для Linux. Установим его для загрузки нашей системы.

Установка GRUB для UEFI-систем 
# Установка GRUB в ESP
grub-install --target=x86_64-efi --efi-directory=/boot/efi --bootloader-id=GRUB

# Генерация конфигурационного файла GRUB
grub-mkconfig -o /boot/grub/grub.cfg
Дополнительно: Включение NetworkManager 
# Включение автоматического запуска NetworkManager
systemctl enable NetworkManager

# Выход из chroot
exit
Совет: Если у вас система с BIOS (не UEFI), используйте команду grub-install --target=i386-pc /dev/sdX (где sdX - ваш диск, например, /dev/sda).

Шаг 8: Завершение установки

Поздравляем! Основная часть установки завершена. Осталось только размонтировать разделы и перезагрузиться.

Завершение установки 
# Размонтирование всех разделов
umount -R /mnt

# Перезагрузка системы
reboot
Важно: После перезагрузки не забудьте извлечь установочный носитель (флешку или диск), иначе система снова запустится с него.

Что делать после установки:

  • Войдите в систему под созданным пользователем
  • Установите графическую оболочку (GNOME, KDE Plasma, XFCE и т.д.)
  • Установите драйверы для вашего оборудования
  • Настройте звук, если он не работает
  • Установите необходимые программы и настройте систему под свои нужды
Совет: Для продолжения настройки после установки ознакомьтесь с официальной Arch Wiki - это лучший источник информации об Arch Linux.

Troubleshooting - Решение проблем

Нет интернета после запуска iwctl

Убедитесь, что ваш Wi-Fi адаптер поддерживается. Для некоторых адаптеров нужны проприетарные драйверы. Проверьте, загружен ли нужный модуль ядра с помощью lsmod | grep wifi.

Ошибка "Failed to commit transaction" при установке пакетов

Попробуйте обновить ключи пакетов: pacman-key --init && pacman-key --populate archlinux. Также проверьте подключение к интернету и попробуйте обновить зеркала с помощью pacman -Syy.

Система не загружается после установки GRUB

Убедитесь, что вы установили GRUB на правильный диск и правильно указали EFI-раздел. Проверьте, поддерживает ли ваша система UEFI и включен ли он в BIOS/UEFI настройках.

Не работает звук после установки

Установите и включите звуковой демон: pacman -S pulseaudio pulseaudio-alsa и systemctl --user enable pulseaudio. Для ALSA: pacman -S alsa-utils и запустите alsamixer для настройки.

Черный экран после установки графической оболочки

Возможно, не установлены драйверы для вашей видеокарты. Установите соответствующие драйверы: nvidia для NVIDIA, xf86-video-amdgpu для AMD, xf86-video-intel для Intel.
